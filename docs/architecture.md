# 技术方案说明

## 能否一次性完成？

可以一次性交付一个可验收的 MVP：
- 具备 PDF 上传、解析、提取、匹配评分、JSON 返回、缓存、前端演示与部署文档。
- 生产级增强（OCR、多语言简历、复杂实体抽取、A/B 评估）可作为下一阶段迭代。

## 架构

- 前端：静态网页（GitHub Pages）
- 后端：FastAPI（可部署阿里云函数计算）
- 缓存：Redis（可选，未配置时回退内存缓存）

流程：
1. 前端上传 PDF -> `/api/resume/upload`
2. 后端解析 PDF 文本并清洗
3. 规则抽取关键字段
4. 存入缓存，返回结构化结果
5. 提交 JD -> `/api/job/match`
6. 关键词+经验打分输出总分

## 评分策略

`total = 0.7 * keyword_match_rate + 0.3 * experience_score`

## 风险与建议

- 扫描版 PDF 文本抽取可能失败，建议后续接入 OCR。
- 规则抽取在格式复杂简历上有误差，建议接入 LLM + schema 校验。
- 大规模并发需引入任务队列与限流。
